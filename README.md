# FlexPass

**Gym memberships you actually own.**

[![CI](https://img.shields.io/badge/ci-green-brightgreen.svg)](https://github.com/Meru143/flexpass/actions)
[![License: MIT](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![Polygon](https://img.shields.io/badge/network-Polygon%20PoS-8247e5.svg)](https://polygon.technology/)

FlexPass is a blockchain-based gym membership portability protocol. It issues time-bound ERC-4907 membership NFTs with ERC-2981 royalties, lets members list unused access on a peer-to-peer marketplace, and settles MATIC payments so gyms receive resale royalties on-chain.

## Architecture

The system separates gym approval, membership ownership, marketplace settlement, indexing, API caching, and verifier integration into different packages. The contracts are the source of truth; the subgraph and API make reads faster for the frontend and kiosk services.

```text
                 +----------------------+
                 |      Next.js dApp    |
                 | dashboard/market/sell|
                 +----------+-----------+
                            |
                            v
+------------+      +-------+--------+      +------------------+
| Gym owners | ---> | Hono API cache | <--- | The Graph Studio |
+------------+      +-------+--------+      +---------+--------+
                            |                         ^
                            v                         |
                 +----------+-----------+             |
                 |     Polygon PoS      |-------------+
                 | GymRegistry          |
                 | GymMembership NFT    |
                 | FlexPassMarket       |
                 +----------+-----------+
                            |
                            v
                 +----------------------+
                 | @flexpass/verifier   |
                 | kiosk access checks  |
                 +----------------------+
```

## Quick Start

Install Foundry, Node.js 22, and npm first. Clone the repository, install each JavaScript package, copy the example environment file, then start the frontend.

```powershell
git clone https://github.com/Meru143/flexpass.git
cd flexpass

cd frontend; npm install; cd ..
cd api; npm install; cd ..
cd subgraph; npm install; cd ..
cd sdk; npm install; cd ..

Copy-Item .env.example .env
make dev-frontend
```

For a local chain, run Anvil in another terminal and seed demo contracts. The `make anvil` target forks Polygon and needs `POLYGON_RPC_URL`; if you do not have an RPC URL, run `anvil --chain-id 31337` directly.

```powershell
anvil --chain-id 31337
make seed
```

## How It Works

FlexPass keeps membership access portable without making gyms operate a marketplace themselves. A gym is registered and approved once, membership NFTs are minted with an expiry timestamp, and the marketplace contract escrows listed NFTs so a resale can settle in one transaction.

```text
Gym registers -> Owner approves gym -> Gym mints NFT
                                      |
                                      v
Member owns time-bound access -> Member lists unused time
                                      |
                                      v
Buyer pays MATIC -> market pays royalty -> market pays seller
                                      |
                                      v
Buyer receives NFT and ERC-4907 user access until original expiry
```

The important contract behavior is in `FlexPassMarket.buyMembership`. It marks the listing inactive before external value transfers, transfers the NFT from escrow to the buyer, restores the ERC-4907 user role with the stored expiry, then pays the gym royalty, protocol fee, and seller proceeds.

## For Gyms

Gyms register through `GymRegistry.registerGym(gymAddress, treasury, name, royaltyBps)`. The protocol owner approves the gym before minting is allowed, which prevents a third party from impersonating a venue and minting memberships against its brand.

Royalties are configured in basis points and enforced inside `FlexPassMarket`. On each marketplace resale, the market calls `royaltyInfo(tokenId, salePrice)`, pays the returned receiver, pays the protocol fee, and sends the remainder to the seller.

## For Members

Members receive a membership NFT whose `userOf(tokenId)` returns their wallet until the expiry timestamp. If they can no longer use the membership, they approve the marketplace, list the token with a MATIC price, and the marketplace escrows it until the listing is bought, repriced, cancelled, or cleaned after expiry.

Buyers purchase the remaining time, not a fresh duration. The original expiry is preserved after resale, so the dApp and verifier show exactly how much access remains before the purchase is confirmed.

## For Kiosk Integrators

Kiosk and gate systems use `@flexpass/verifier` to read access state directly from the deployed `GymMembership` contract. The SDK returns a typed access decision with the current user wallet, expiry time, tier, and gym address.

```powershell
npm install @flexpass/verifier viem
```

```ts
import { createVerifier } from "@flexpass/verifier";
import gymMembershipAbi from "./abis/GymMembership.json";

const verifier = createVerifier({
  rpcUrl: process.env.POLYGON_RPC_URL!,
  contractAddress: "0x465CF3a5918534d94BA62F3A7980f5ffB0277168",
  abi: gymMembershipAbi,
  offlineGracePeriodMs: 60_000,
});

const result = await verifier.checkAccess(1);

if (result.valid) {
  console.log("Access granted until", result.expiresAt.toISOString());
} else {
  console.log("Access denied");
}
```
